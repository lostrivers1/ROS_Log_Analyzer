# ROS_Debug_Agent

ROS 导航系统日志自动化分析与 Debug 智能代理

## 项目概述
本项目构建了一个 **基于 AI/Agent 的 ROS 导航系统自动化日志分析与 Debug Agent**，用于实时收集、解析和诊断 ROS 导航栈日志，自动发现异常和性能瓶颈。该 Agent 能显著提升团队在 ROS 系统调试与优化过程中的效率。

---

## 核心功能

1. **自动化日志收集**
   - 实时订阅 ROS 日志节点（如 `/rosout`、`/move_base`）。
   - 将日志解析为结构化数据，方便后续分析。
   - 支持长期存储和批量分析。

2. **智能异常检测**
   - 基于规则和机器学习算法，自动识别路径规划失败、局部避障异常、SLAM 漂移、传感器异常等问题。
   - 提供根因分析和优先级排序，快速定位问题。

3. **多 Agent 协作分析**
   - 多 Agent 并行处理日志数据，实现跨模块异常关联分析。
   - 自动生成 Debug 报告，包括问题描述、日志片段和优化建议。
   - 可生成 PR 或优化文档，直接指导开发与调试。

4. **自动化回归验证**
   - 在 Gazebo 仿真环境中自动复现问题。
   - 验证优化策略是否有效，减少人工测试成本。

---

## 技术优势
- **效率提升**：在团队落地测试中，将问题定位效率提升约 70%。
- **可扩展性**：模块化设计，兼容 ROS 1/ROS 2 及多机器人系统。
- **高自动化**：从日志采集、分析到报告生成几乎无需人工干预。
- **数据驱动优化**：利用历史日志数据训练模型，实现动态优化策略。

---

## 项目落地
- 已在 5 人研发团队中使用，每日分析约 10,000 条日志。
- 自动生成报告帮助团队快速定位导航异常，减少人工调试时间。
- 可扩展至更大规模机器人团队和复杂导航场景。

---

## 使用说明
1. 启动 ROS Master。
2. 运行 `src/log_collector.py` 收集日志。
3. 运行 `src/debug_agent.py` 自动分析并生成报告。
